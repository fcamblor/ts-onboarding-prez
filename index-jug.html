Amorce

  • Plus d'un an que je fais du Typescript au quotidien
  • Partager mon expérience sur la mise en place de TS sur un projet existant.
  • REX projet 4SH > 6 mois : ça se passe bien et cela apporte énormément de "meta-information" / doc sur le front.

4 Parties

  1. Mise en place de Typescript sur projet existant
  2. Minimum vital à connaître pour faciliter votre progression
  3. Migration d'une première classe JS en TS
  4. Cohabitation modules RequireJS et AngularJS

Setup

  • npm install --save-dev typescript
  • Création tsconfig
  • Explication process de compilation
  • Remplacement dans le index.html & run

Tricks

  • Live template "TSConfig" ou "tsconfigcontent"
  • Paramétrage d'Intellij pour compilation TS

Sandbox

  • Création fichier Sandbox.ts
  • Rapide prez des Classes
  • Code généré
  • Rapide prez des Interfaces
  • Code non généré
  • Typage structurel
  • Imports/Exports
  • Code généré + rajout requirejs à la fin

Tricks

  • Live template "tshello"
  • "Espaces" de déclaration : variables et types. Parallèle avec Classes & interfaces.
  • SourceMaps dans le browser
  • Fonctionnement du inlineSources
  • instanceof d'une classe / interface

Interop

  • Renommage WorkshopLogger.js => ts. Migrer les feuilles dans l'arbre de dépendances.
  • Désactivation du implicit any pour montrer les niveaux de compil
  • Présentation du declare var de type any (workaround rapide)
  • Présentation fichiers .d.ts & @types
  • Installation @types : npm install --save @types/lodash@3.10.1 @types/handlebars@4.0.29 @types/angular@1.5.8
  • Ecriture d'un typings manuel pour stacktrace-js

Tricks

  • Rechercher un @type : http://aka.ms/types
  • Lister les versions d'un d.ts : npm show @types/handlebars
  • Live template IntelliJ "printstacktrace"

Fin refacto Logger

  • Classe avec méthodes public static
  • Interfaces LogEventCallbackStd|UserParam
  • Type LogEvent qui ressemble a un enum
  • LogEventObserver qui est un object literal
  • Externalisation de la classe Logger pour faciliter son export futur

Tricks

  • Petit focus sur les enums qui sont vraiment pourris en TS
  • let est mieux que var

NG-Controller en TS

  • Classe controller
  • Injection de dépendances via public static $inject
  • Possibilité de colocaliser le template avec le controller

Tricks

  • Fat arrows ne redéfinissent pas this, super, arguments et target.new

Export d'un module

  • Exporter la version et commenter l'injection dans app.js
  • Dans console : 
    require(['utils/Version'], function(){ console.log("Version loaded"); })
  • Déclaration module pour angular
  • Migration app.js => app.ts + Import de la constante
  • Suppression ng-app + Migration de tous les composants angular en TS + déclaration de chacun d'eux dans le app.ts
  • Remplacer logger injecté via angular par l'appel static à la classe WorkshopLogger

Tricks

  • Live template "angularrequire"
  • Le controller peut implémenter ng.IController
  • $http peut être défini via un IHttpService
  • Le component peut implémenter ng.IDirective
